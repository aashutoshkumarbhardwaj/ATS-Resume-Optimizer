═════════════════════════════════════════════════════════════════════════════════
                            DATA FLOW ARCHITECTURE
═════════════════════════════════════════════════════════════════════════════════

APPLICATION TRACKING FLOW
Extension (Autofill)
    ↓ Records application
chrome.storage.local (applicationHistory)
    ↓ If Job Orbit connected & auto-sync enabled
handleAutofillResults() → syncJobTrackingWithOrbit()
    ↓
Job Orbit API (via backend)
    ↓
User sees ✅ "Synced X applications"

JOB TRACKER DISPLAY FLOW
Click "Job Tracker" tab
    ↓ switchTab('jobtracking')
loadJobTracking()
    ↓ Get applicationHistory from storage
displayJobTracking(applications)
    ↓
Render list with company, job, location, salary
    ↓
User sees all tracked applications

SYNC WORKFLOW
User clicks "Sync with Job Orbit"
    ↓
syncJobTrackingWithOrbit()
    ↓
Get API key from chrome.storage.sync
Get applications from chrome.storage.local
    ↓
POST /api/job-orbit/sync
    ↓
Backend: JobOrbitService.syncApplications()
    ↓
Call Job Orbit API
    ↓
Return results: { totalApplications, successCount, results }
    ↓
Update UI: "✅ Synced 10 applications to Job Orbit!"
